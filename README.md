Parte 1 – Construcción del MER del Proyecto Integrador

Aplicar los conceptos vistos en el entrenamiento sobre Modelo Entidad-Relación (MER) para modelar correctamente la base de datos del proyecto integrador en el que actualmente están trabajando.

Cada equipo deberá:
Analizar su proyecto integrador.
Identificar correctamente:

Entidades principales
Atributos
Claves primarias (PK)
Claves foráneas (FK)
Relaciones
Cardinalidades (1:1, 1:N, N:M)

Construir el Modelo Entidad-Relación (MER) completo.
¿Qué deben tener en cuenta?
El MER debe:
Tener nombres claros y coherentes.
No duplicar información innecesariamente.
Definir correctamente las relaciones.
Identificar relaciones muchos a muchos y resolverlas correctamente (tabla intermedia).
Aplicar buenas prácticas de modelado.

Entregable

Subir:
Archivo PDF o Imagen clara del diagrama

Debe ser legible y bien organizado.

Parte 2 – Plan de Estudio Obligatorio (Estudiar Fuerte)

Además del MER, deben estudiar profundamente los siguientes temas, ya que serán fundamentales para lo que viene en el modulo.

ESTOS TEMAS NO SON NEGOCIABLES.

Si no los dominan, van a estar perdidos en la siguiente etapa.


| Paso | Tema                             | Qué deben hacer                                                                                              | Meta de aprendizaje                                                    |
| ---- | -------------------------------- | ------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------- |
| 1    | Introducción a SQL               | Comprender qué es SQL, para qué sirve, qué es una base de datos relacional y cómo se conecta con el MER.     | Entender que SQL es el lenguaje que convierte el MER en tablas reales. |
| 2    | Tipos de Comandos SQL            | Diferenciar DDL, DML, DQL, DCL y TCL, identificando qué hace cada uno.                                       | Saber clasificar cualquier comando SQL correctamente.                  |
| 3    | DDL – Creación de Estructuras    | Practicar CREATE, ALTER y DROP. Estudiar tipos de datos y restricciones (PK, FK, NOT NULL, UNIQUE, DEFAULT). | Poder crear en SQL las tablas diseñadas en el MER.                     |
| 4    | DML – Manipulación de Datos      | Practicar INSERT, UPDATE y DELETE usando correctamente WHERE.                                                | Insertar y modificar datos sin afectar registros incorrectos.          |
| 5    | Consultas Básicas (SELECT)       | Practicar SELECT, FROM y WHERE con distintos filtros.                                                        | Consultar información específica de manera correcta.                   |
| 6    | Filtrado y Ordenamiento Avanzado | Estudiar ORDER BY, LIMIT y operadores AND, OR, NOT, BETWEEN, LIKE.                                           | Refinar y organizar resultados profesionalmente.                       |
| 7    | Funciones de Agregación          | Practicar COUNT, SUM, AVG, MIN y MAX.                                                                        | Realizar análisis básicos sobre los datos.                             |
| 8    | Agrupación de Datos              | Estudiar GROUP BY y HAVING, entendiendo su diferencia con WHERE.                                             | Generar reportes y consultas agrupadas correctamente.                  |
| 9    | Relaciones y JOINS               | Practicar INNER JOIN, LEFT JOIN y RIGHT JOIN usando claves foráneas.                                         | Consultar información combinando múltiples tablas.                     |
